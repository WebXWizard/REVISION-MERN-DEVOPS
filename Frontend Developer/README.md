# Frontend Developer Revision

This folder is a structured Frontend Developer learning path for building responsive, accessible, performant, and production-ready user interfaces.

Use it like this:

1. Start with `00-Frontend-Plan`.
2. Read each folder's `Study.md`.
3. Revise topic-wise files.
4. Practice `Interview-questions.md`.
5. Build the projects in `12-Frontend-Projects`.
6. Use `14-Rapid-Revision` before interviews.

Daily routine:

```txt
45 min concepts
60 min UI or JavaScript coding
45 min project work
30 min interview questions
15 min project explanation
```

Frontend answer formula:

```txt
User need -> Component design -> State -> API/data flow -> Accessibility -> Performance -> Tradeoff
```

Core goal:

```txt
Learn how to build clean UI, manage state, call APIs, test behavior, optimize performance, and explain frontend decisions.
```
