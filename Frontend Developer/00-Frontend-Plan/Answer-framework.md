# Frontend Answer Framework

Use this format for frontend interview answers:

```txt
What it is -> Why it matters -> Example in UI -> Implementation detail -> Common mistake -> Best practice
```

For project questions:

```txt
Problem -> Users -> Pages/components -> State -> API flow -> Accessibility -> Performance -> Deployment -> Improvement
```

Good frontend answers include:

- Component structure
- State/data flow
- User experience
- Accessibility
- Performance
- Error and loading states
