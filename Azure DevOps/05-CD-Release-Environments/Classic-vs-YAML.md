# Classic Release Vs YAML

Classic release:

- UI-based release pipeline.
- Useful in older Azure DevOps setups.

YAML pipeline:

- Pipeline as code.
- Versioned with repository.
- Easier to review.
- Better for modern CI/CD.

Interview line:

```txt
Classic releases are UI-driven, while YAML pipelines keep CI/CD definitions in source control.
```
