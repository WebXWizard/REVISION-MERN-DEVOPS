# Last Minute Notes

Remember:

- Protect main branch with PR and build validation.
- Keep pipeline YAML in source control.
- Use stages for build, test, and deploy.
- Publish artifacts from CI and deploy the same artifact.
- Use service connections with least privilege.
- Store secrets in Key Vault or secure variables.
- Add approvals before production.
- Tag Docker images with build ID or commit SHA.
- Use immutable image tags for deployment.
- Monitor deployment with logs, metrics, and smoke tests.
- Always mention rollback.
