# Last Minute Notes

Remember:

- Backend must validate input and enforce permissions.
- Use correct status codes.
- Never store plain passwords.
- Secrets should not be committed.
- Database indexes improve reads but cost writes.
- Caching can create stale data.
- Queues are good for slow background work.
- Monitoring should track latency, traffic, errors, and saturation.
- Project answers should include schema, APIs, auth, deployment, and challenges.
