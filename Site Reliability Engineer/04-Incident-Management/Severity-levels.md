# Severity Levels

Example severity model:

```txt
SEV1: Major outage, many users blocked, urgent response.
SEV2: Significant degradation, important feature broken.
SEV3: Partial issue, workaround available.
SEV4: Minor issue, low user impact.
```

Severity should consider:

- Number of users affected
- Revenue or business impact
- Data loss risk
- Security impact
- Duration
- Workaround availability
