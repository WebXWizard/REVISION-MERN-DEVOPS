# Reliability Culture

Good reliability culture means teams learn from failure and improve systems.

Important habits:

- Write blameless postmortems.
- Use evidence instead of guesses.
- Measure reliability with SLOs.
- Prefer simple systems where possible.
- Make failure visible early.
- Share ownership across development and operations.

Bad signs:

- Alerts are ignored.
- Incidents repeat.
- Runbooks are missing.
- Deployments are scary.
- Only one person understands production.
