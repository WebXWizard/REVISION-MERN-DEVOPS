# Site Reliability Engineer Revision

This folder is a structured SRE learning path for moving from DevOps foundations to production reliability work.

Use it like this:

1. Start with `00-SRE-Plan`.
2. Read each folder's `Study.md`.
3. Revise the focused topic files.
4. Practice `Interview-questions.md`.
5. Build the projects in `11-SRE-Projects`.
6. Use `13-Rapid-Revision` before interviews.

Daily routine:

```txt
45 min concept revision
45 min commands and troubleshooting
45 min project or lab work
30 min interview questions
15 min writing a runbook, postmortem, or SLO explanation
```

SRE answer formula:

```txt
User impact -> Signal -> Diagnosis -> Mitigation -> Prevention -> Measurement
```

Core goal:

```txt
Learn how to keep systems reliable, observable, scalable, secure, and recoverable in production.
```
