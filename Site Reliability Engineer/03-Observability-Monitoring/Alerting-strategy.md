# Alerting Strategy

Good alerts should be:

- Actionable
- Related to user impact
- Clear about severity
- Linked to a runbook
- Low noise

Bad alerts:

- Fire too often
- Do not need action
- Alert on harmless internal symptoms
- Lack context
- Wake people without user impact

Useful alert types:

- High error rate
- High p95 latency
- Service unavailable
- Error budget burn rate
- Disk almost full
- Certificate expiry
