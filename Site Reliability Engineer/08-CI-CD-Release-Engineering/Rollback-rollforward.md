# Rollback And Rollforward

Rollback:

- Return to previous known-good version.
- Useful when fast recovery is needed.

Rollforward:

- Deploy a fix on top of the bad release.
- Useful when rollback is not possible because of data or schema changes.

Before release:

- Know rollback command.
- Confirm database migration strategy.
- Monitor key metrics.
- Keep release notes.
