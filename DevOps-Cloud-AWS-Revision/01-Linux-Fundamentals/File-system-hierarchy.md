# File System Hierarchy

Linux directories have standard purposes.

Important paths:

- `/etc`: configuration files
- `/var`: variable data like logs
- `/home`: user directories
- `/bin`: essential binaries
- `/usr`: user programs and libraries
- `/tmp`: temporary files

Real-world example:

- Nginx config is usually under `/etc/nginx`.
- Logs are commonly under `/var/log`.

Common mistakes:

- Editing config without backup.
- Deleting files from system directories blindly.
