# SSH Hardening

SSH hardening remote server access secure karta hai.

Good practices:

- Use key-based login.
- Disable root login.
- Restrict port 22 to your IP.
- Keep private key safe.

Common mistakes:

- Sharing private key.
- Opening SSH to `0.0.0.0/0` in production.
