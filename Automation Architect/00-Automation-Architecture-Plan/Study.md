# Automation Architect Study Plan

An Automation Architect designs automation systems, not just scripts.

Focus areas:

- Understand business process and pain.
- Select good automation candidates.
- Design reusable automation patterns.
- Integrate systems through APIs and events.
- Add security, audit, and approval controls.
- Monitor automation health.
- Measure time saved, errors reduced, and cost impact.

Suggested order:

1. Automation fundamentals
2. Process discovery and assessment
3. Scripting and reusable libraries
4. Workflow orchestration
5. API and event-driven integration
6. Infrastructure automation
7. CI/CD and release automation
8. Cloud and platform automation
9. RPA and low-code automation
10. Testing automation
11. Observability and operations automation
12. Security and governance
13. Reliability, scalability, and cost
14. Projects and mock interviews
