# Platform Engineer Answer Framework

Use this format for interview answers:

```txt
Developer pain -> Reusable platform solution -> Workflow -> Guardrails -> Automation -> Metrics -> Improvement
```

Example:

```txt
Developers were spending too much time creating services manually. I would build a golden path with a service template, repository creation, CI/CD pipeline, Kubernetes deployment, observability defaults, security policies, and documentation. Success would be measured by onboarding time, deployment frequency, failure rate, and developer feedback.
```

Good answers include:

- Developer problem
- Platform capability
- Self-service design
- Security guardrails
- Automation
- Observability
- Adoption metrics
- Tradeoffs
