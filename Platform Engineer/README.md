# Platform Engineer Revision

This folder is a structured Platform Engineer learning path for building internal platforms, improving developer experience, automating infrastructure, and creating secure self-service delivery systems.

Use it like this:

1. Start with `00-Platform-Engineering-Plan`.
2. Read each folder's `Study.md`.
3. Revise topic-wise files.
4. Practice `Interview-questions.md`.
5. Build the projects in `13-Platform-Projects`.
6. Use `15-Rapid-Revision` before interviews.

Daily routine:

```txt
45 min platform concepts
45 min Kubernetes, IaC, or CI/CD practice
45 min project or lab work
30 min interview questions
15 min explaining developer experience and platform value
```

Platform Engineer answer formula:

```txt
Developer problem -> Platform capability -> Self-service flow -> Guardrails -> Automation -> Observability -> Outcome
```

Core goal:

```txt
Learn how to build reliable internal platforms that help developers ship software safely, quickly, and consistently.
```
