# Security Engineering Study Plan

Security Engineering is about reducing risk while keeping systems usable.

Focus areas:

- Understand assets and threats.
- Apply defense in depth.
- Secure identities and access.
- Harden systems and networks.
- Find and fix vulnerabilities.
- Detect suspicious behavior.
- Respond to incidents.
- Build secure-by-default delivery workflows.

Suggested order:

1. Security fundamentals
2. Network security
3. Linux and endpoint security
4. Cloud security and IAM
5. Web and API security
6. DevSecOps
7. SOC and incident response
8. Vulnerability management
9. Secrets and cryptography
10. Kubernetes and container security
11. Compliance and GRC
12. Projects and mock interviews
