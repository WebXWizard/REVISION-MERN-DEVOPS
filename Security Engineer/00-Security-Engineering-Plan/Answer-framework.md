# Security Answer Framework

Use this format for security interview answers:

```txt
What is protected -> What can go wrong -> How likely and severe -> Preventive control -> Detective control -> Response plan -> Improvement
```

Example:

```txt
For a public API, I would protect user data by using authentication, authorization, input validation, rate limiting, HTTPS, secure logging, and monitoring. I would detect suspicious behavior through logs and alerts, then respond with token revocation, blocking, investigation, and post-incident remediation.
```

Good answers include:

- Asset
- Threat
- Risk
- Control
- Detection
- Response
- Tradeoff
