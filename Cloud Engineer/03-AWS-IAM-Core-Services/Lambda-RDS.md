# Lambda And RDS

Lambda:

- Serverless compute.
- Runs code in response to events.
- No server management.

RDS:

- Managed relational database service.
- Supports engines like MySQL, PostgreSQL, and MariaDB.

Use Lambda for:

- Event processing
- Scheduled jobs
- Lightweight APIs

Use RDS for:

- Structured relational data
- Transactions
- SQL workloads
