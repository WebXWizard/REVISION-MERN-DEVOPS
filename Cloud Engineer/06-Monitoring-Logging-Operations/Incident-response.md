# Incident Response

Incident response is the process of restoring service during failures.

Basic flow:

1. Detect alert.
2. Check user impact.
3. Triage severity.
4. Mitigate.
5. Communicate.
6. Resolve.
7. Document root cause and prevention.

Cloud signals:

- CloudWatch alarms
- Load balancer target health
- Recent deployments
- CloudTrail events
- Service Health Dashboard
