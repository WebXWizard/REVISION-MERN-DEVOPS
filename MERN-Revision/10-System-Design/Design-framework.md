# Design Framework

Use this framework for any fresher-level system design.

Steps:

```txt
1. Requirements
2. Users and roles
3. Main entities
4. API design
5. Database schema
6. Auth and permissions
7. Scaling basics
8. Edge cases
```

Interview tip:

- Start simple, then add improvements.

Common mistakes:

- Direct database design start karna without requirements.

Mini task:

- Apply this framework to a job portal.
