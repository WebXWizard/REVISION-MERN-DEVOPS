# Schema Design

Schema design decide karta hai data ka structure kaisa hoga.

Good design considers:

- Query patterns
- Data size
- Relationships
- Indexes
- Validation

Real-world example:

- E-commerce: User, Product, Cart, Order.

Common mistakes:

- Query pattern soche bina schema banana.

Interview questions:

- How do you design MongoDB schema?
- Embed vs reference decision?

- Design User, Product, Cart, Order schemas.
