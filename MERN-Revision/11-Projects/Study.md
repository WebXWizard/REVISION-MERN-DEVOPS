# Projects Study Notes

Your projects are your strongest fresher proof. Interviewer checks whether you really built and understood them.

Prepare every project using this structure:

```txt
1. Problem statement
2. Tech stack
3. Architecture
4. Main features
5. Database schema
6. API flow
7. Authentication flow
8. Biggest challenge
9. What you learned
10. How you would improve/scale
```

Common MERN project explanation:

```txt
I built this project to solve...
Frontend is built with React/Next.js.
Backend APIs are built with Node.js and Express.
MongoDB is used because data is document-based and flexible.
Authentication is handled using JWT/cookies.
The biggest challenge was...
```

Common mistakes:

- Saying "I used MongoDB because it is easy" only.
- Not knowing your own APIs.
- Not knowing schema fields.
- Not explaining tradeoffs.
- No deployment understanding.
