# Why MongoDB

Good fresher answer:

```txt
I used MongoDB because my project data is JSON-like and flexible. It works naturally with JavaScript objects in MERN. For example, products/orders/users can be stored as documents, and Mongoose helps with schema validation.
```

Avoid:

- "Because it is easy" only.
- "MongoDB is always better than SQL."

Interview questions:

- Why MongoDB?
- Why not SQL?
- How did you design schema?
