# React Interview Questions

Concept questions:

1. What is React?
2. Why use React?
3. Component vs element.
4. Props vs state.
5. What is JSX?
6. What is Virtual DOM?
7. What is reconciliation?
8. Why are keys important?
9. Controlled vs uncontrolled components.
10. Explain `useEffect`.
11. `useMemo` vs `useCallback`.
12. When to use `useRef`?
13. Context API vs Redux.
14. What is prop drilling?
15. What causes re-rendering?

Debugging exercises:

1. State update is not reflected because array is mutated directly.
2. `useEffect` keeps running infinitely.
3. List UI behaves incorrectly because index is used as key.
4. Child component re-renders unnecessarily.

Component design questions:

1. Design a reusable Button component.
2. Design a search and filter component.
3. Design a protected route.
4. Design a form component with validation.

Scenario questions:

1. API data should load when page opens. Which hook?
2. Need to avoid expensive recalculation. What will you use?
3. Need to access DOM input directly. What will you use?
4. App-wide user data needed. Context or Redux?
