# Component Design

Good component design reusable, readable, and testable hota hai.

Principles:

- Single responsibility
- Clear props
- Composition
- Avoid unnecessary global state
- Keep UI and business logic clean

Common examples:

- Button
- Input
- Modal
- Card
- DataTable

Interview questions:

- How would you design reusable Button?
- How do you decide component boundaries?

Mini task:

- Design reusable Modal component with open, close, title, children.
